Obtain help

To obtain help on available commands use the help command. 

default> help

Commands

The RDF Coder command line provides a way to handle code models.
A code model (for semplicity model) is a representation of a class path that can be composed by source code (.java), compiled code(.class), documentation (.java documentation) and other supported formats. A model can be queried by SPARQL queries, or browsed via inspection.

A model life cycle starts with a from scratch creation or by loading the model from a storage. It is possible to populate a model by loading classpath objects. The loading of classpath objects can be done is different sessions. A model can be cleaned to remove the entire content. A populated model can be queried by using SPARQL, on inspected with inspector functions. Finally a model can be made persistent on any supported code storage like database or filesystem.
Many supported commands are specifically applied on a model, at any time the console has a selected model that is printed in prompt. If a model related command is invoked with no explicit model name, the command is invoked on the selected one.

The main commands are:

debug Enables / disables the debug mode.
debug [true | false]?
With argument this command enables / disables the debug mode. If called without arguments shows the current value of the debug flag.
pwd Prints the current directory.
pwd
This command prints the current working directory, such directory is used every time a relative path is specified as argument in other commands requiring file system resources.
cd Changes the current directory.
cd <path_to_dir>
This command changes the current working directory. The <path_to_dir> argument can be both an absolute path or a relative path.
ls Lists the content of the current directory.
ls
This command lists the content of the current working directory, reporting for every file the type (file or directory), the size and the writing permissions.
list Lists all the currently loaded models.
list
prints out the list of all the loaded models, the name of the selected model is marked with an asterisk.
setmodel Allows to set the selected model.
setmodel [<model_name>]
model name is the name of the model to set as selected, if not specified the command prints out the name of the currently selected model.
Any command requiring a model name uses the selected model if no explicit model was specified. The selected model name is printed out in prompt.
newmodel Creates a new Code Model to store library informations.
newmodel <model_name>
model name is the name of the new model, that must differ from the names of the already loaded models.
This command creates a code model from scrath. A CodeModel is a triples container able to handle data generated by code parsers. This model can be queried and inspected, stored and relaoded from any supported storage.
clearmodel Cleans the content of a CodeModel.
clearmodel [<model_name>]
model name is the name of the model model to delete, if not specified the selected model is deleted.
This command removes all the triples present in the specified model.
removemodel Removes a previously created Code Model.
removemodel [<model_name>]
model name is the name of the model to remove, if not specified the selected modelis removed.
Removing a model deletes all the not stored content. To make the content of the model persistent use savemodel.
querymodel Allows to perform a query (SPARQL) on the current model.
querymodel <query>
Performs the SPARQL query on the selected model.
inspectmodel Allows to inspect the current model.
inspectmodel <query>
Performs an inpection query on the selected model. TODO: provide references to the query syntax.
loadmodel Loads a library model.
loadmodel <storage_name> <storage_param_1> ...
This command loads a model with name storage_name retrieving it from a phisical storage which access is described using a list of storage_parameters.
savemodel Saves a model on a storage.
savemodel <storage_type> <storage_parameters>
Saves the model on the storage specified with storage type using the given parameters. Storage type can be fs or db. The parameters are storage dependend. For filesystem there is fsfile containing the path to the file to store the model. For db there are dbuser, dbpassword, dbname. TODO: TBC
loadclasspath Loads source classpath extracting the RDF representation.
loadclasspath [classpath]
Loads the classpath on the selected model. The classpath is a : separated list of paths locating source packages, class packages or jar files. If paths are expressed as relative ones, the reference dir is given by the pwd command.
