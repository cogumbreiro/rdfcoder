Obtain help

To obtain help on available commands use the help command. 

default> help

Commands

The RDF Coder command line provides a way to handle code models.
A code model (for semplicity model) is a representation of a class path that can be composed by source code (.java), compiled code(.class), documentation (.java documentation) and other supported formats. A model can be queried by SPARQL queries, or browsed via inspection.

A model life cycle starts with a from scratch creation or by loading the model from a storage. It is possible to populate a model by loading classpath objects. The loading of classpath objects can be done is different sessions. A model can be cleaned to remove the entire content. A populated model can be queried by using SPARQL, on inspected with inspector functions. Finally a model can be made persistent on any supported code storage like database or filesystem.
Many supported commands are specifically applied on a model, at any time the console has a default model that is printed in the prompt. If a model related command is invoked with no explicit model name, the command is invoked on the default one.

The main commands are:

newmodel Creates a new CodeModel to store library informations.
newmodel <modelname>
modelname is the name of the new model, that must differ from the names of the already loaded models.
This command creates a code model from scrath. A CodeModel is a triples container able to handle data generated by code parsers. This model can be queried and inspected, stored and relaoded from any supported storage.
clearmodel Cleans the content of a CodeModel.
clearmodel [<modelname>]
modelname is the name of the model model to delete, if not specified the default model is deleted.
This command removes all the triples present in the specified model.
removemodel Removes a previously created CodeModel.
removemodel [<modelname>]
modelname is the name of the model to remove, if not specified the default model is removed.
Removing a model deletes all the not stored content. To make the content of the model persistent use savemodel.
setmodel Allows to change the selected model.
setmodel [<modelname>]
modelname is the name of the model to set as default, if not specified the command prints out the name of the default model.
Any command requiring a model name uses the default model name if no explicit model was specified. The default model name is printed in the prompt.
list Lists all the currently loaded models.
list
prints out the list of all the loaded models, the name of the default model is marked with an asterisk.
querymodel Allows to perform a query (SPARQL) on the current model.
querymodel <query>
Performs the SPARQL query on the default model.
load Loads a library on the selected model.
load [classpath]
Loads the classpath on the selected model. The classpath is a : separated list of paths locating source directories, class directories or jar files. If paths are expressed as relative ones, the reference dir is given by the pwd command.
savemodel Saves a model on a storage.
savemodel <storage type> <storage parameters>
