Install RDFCoder

Download the latest distribution of RDFCoder from the download page.
Extract the archive rdf-coder-x.y.z.tgz with the command: 

$ tar xvzf rdf-coder-x.y.z.tgz
Access the extracted folder: 
$ cd rdf-coder-x.y.z
Launch the bash script: 
$ bin/rdfcoder

The Command Line

RDFCoder provides a useful Command Line interactive console.
The complete command line documentation can be found here.
To obtain help on available commands type help.

The main commands are:

newmodelcreates a new CodeModel to store library informations.
clearmodelcleans the content of a CodeModel.
removemodelremoves a previously created CodeModel.
setmodelallows to change the selected model.
listlists all the currently loaded models.
querymodelallows to perform a query (SPARQL) on the current model.
loadloads a library on the selected model.
savemodelsaves a model on a storage.

Create Model

To create the model simply use the command newmodel and set it as the selected model using setmodel.
default > newmodel test
Created model: test
default > setmodel test
default model: test
test > load /path/to/yourlib.jar
loading /path/to/yourlib.jar ... done
[Statistics/Error output]
test > load /path/to/java/src/
loading /path/to/java/src/ ... done
[Statistics/Error output]

Store Model

To store the selected model use the command savemodel. The model can be stored either on the Filesystem or a database storage or any kind of persistent storage.
test > savemodel "fsfilename=/path/to/filename.rdf"
stored model "test" in "/path/to/filename.rdf"

Query Model

To query the selected model use the command querymodel by providing a SPARQL query.
test > querymodel "select ?a ?b ?c where {?a ?b ?c}"
-------------
| a | b | c |
[query result]
-------------
